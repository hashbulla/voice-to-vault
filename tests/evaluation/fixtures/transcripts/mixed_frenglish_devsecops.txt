En fait le issue c'est que le team a mergé une PR sans attendre le CI green, et maintenant le desired state dans le Git repo ne match plus le live state du cluster. J'ai rollback manuellement via kubectl rollout undo mais c'est pas idéal. Pour le secret management, on devrait migrer vers External Secrets Operator plutôt que de hardcoder les secrets dans les values files. Je vais créer un ticket Jira pour tracker tout ça et planifier une session avec l'équipe platform.
